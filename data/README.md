# Audio Data Directory

This directory contains the audio dataset used for training the keyword spotting model.

## Structure

```
data/
├── raw/                    # Raw audio recordings
│   ├── noise/              # Background noise samples
│   ├── silence/            # Silence samples  
│   ├── similar/            # Similar-sounding words (e.g., "stop", "star")
│   ├── speech/             # General speech samples
│   ├── start/              # "start" keyword samples (target class)
│   └── words/              # Other unrelated words
└── processed/              # Extracted features (generated by make_features.py)
    ├── features.npy        # MFCC feature vectors (N x 257)
    ├── labels.npy          # Binary labels: 1="start", 0=other
    ├── labels_actual.npy   # Original label types before binarization
    ├── label_types.npy     # Label category mapping
    └── filenames.npy       # Source audio filenames
```

## Audio Format

- **Sample rate:** 16 kHz
- **Bit depth:** 16-bit PCM
- **Channels:** Mono
- **Format:** WAV

## Feature Extraction

Features are extracted using `python/make_features.py`:

```bash
python python/make_features.py
```

This generates MFCC features (257 coefficients per sample) saved in `processed/`.

## Data Collection

To add new audio samples:

1. Record audio at 16 kHz, mono, WAV format
2. Place files in appropriate `raw/` subdirectory
3. Re-run feature extraction

See `python/collect_data.py` for automated recording scripts.

## Dataset Statistics

- Total samples: Varies (see `features.npy` shape)
- Classes: Binary (start vs. not-start)
- Feature dimension: 257 (MFCC coefficients)
- Train/test split: 80/20 (configured in `train_model.py`)

Two main directories:
- `preprocessed/` - Put your raw recordings here from Audacity or other sources
  - `start/` - Raw recordings of "start" keyword
  - `silence/` - Raw recordings of silence
  - `noise/` - Raw recordings of background noise
  
- `processed/` - Standardized WAV files (16 kHz, 16-bit PCM, mono)
  - `start/` - Processed "start" keyword recordings
  - `silence/` - Processed silence recordings
  - `noise/` - Processed noise recordings

Use zero-padded filenames like `start_0001.wav`. The `python/batch_convert.py` script can convert all WAVs from preprocessed/ to processed/ automatically.

The `processed/` folder is read by the Python model training scripts in `python/`.
